# Hands Heart Design Direction

## 1. Purpose

This document defines the visual north star for the Hands Heart website. It exists so agents can make interface and layout decisions that stay close to the approved reference direction instead of drifting into a generic ecommerce template.

The reference images in `assets/reference/` are proprietary project reference assets. The entire folder is the true visual north star for the initial build and should be matched as closely as practical in layout rhythm, color energy, typography feel, illustration placement, product presentation, and interaction styling.

Agents must not copy, export, redistribute, or use the reference images themselves as production site assets. They are local design references only.

## 3. Design Goal

The Hands Heart website should feel like a joyful illustrated lifestyle storefront built around a simple symbolic drawing. It should be bold, friendly, graphic, and memorable while still making products easy to browse and buy.

The site should not feel like a restrained gallery anymore. The reference direction is more playful: cream paper-like space, strong black outlines, rounded containers, expressive display type, bright color blocks, and repeated hand-symbol moments.

The finished site should feel close to the reference images while using Hands Heart assets, copy, products, and Shopify data.

---
## 4. Visual Principles

**Symbol First**
- The heart hands drawing is the core visual identity.
- It should appear prominently in the hero and recur throughout the experience.
- Do not reduce it to a tiny logo-only treatment.

**Playful Structure**
- Use big rounded sections, pill-shaped controls, thick outlines, and confident spacing.
- The page should feel intentionally composed, not like stacked generic cards.

**Cream Canvas**
- The base background should be warm off-white or cream rather than pure white.
- Bright colors should sit on top of the cream canvas as graphic shapes, panels, badges, or product tile backgrounds.

**Bold Black Linework**
- Use black outlines as a major visual device.
- Header, buttons, product labels, filters, cards, and decorative shapes can use thick black borders.

**Candy Color Accents**
- Use bright blue, pink, yellow, orange, lavender, green, and soft sky blue as accent colors.
- Colors should feel playful and saturated, but the cream canvas and black type should keep the site readable.

**Commerce Clarity**
- Product cards can be playful, but product name, price, variant state, and add-to-cart actions must remain clear.
- Do not let decorative shapes obscure commerce actions.

---
## 5. Layout Direction

**Header**
- Use a large rounded pill header with black outline.
- Include logo/mark at left, navigation in the middle, and cart/bag control on the right.
- The cart/bag control should feel like a distinct pill button.
- Active nav state may use a small underline or accent mark.

**Hero**
- Use oversized display type on the left.
- Place a large heart hands illustration on the right inside a rounded outlined frame or colorful rounded shape.
- Include a clear primary CTA such as "Shop Collection."
- Decorative color shapes may frame the hero but should not block text or CTA.

**Featured Products**
- Use product tiles with large rounded color backgrounds.
- Product image should dominate the tile.
- Product name and price should sit in an outlined label area near the bottom.
- Action affordance may be a small circular arrow button.

**About / Story**
- Use a large rounded color panel with bold headline copy.
- Pair text with the heart hands illustration or related graphic treatment.
- Keep the story short and visually strong.
- Longer story pages may use large rounded mixed-media panels, icon cards, and bright horizontal statement bands as shown in `assets/reference/about.png`.

**Shop Page**
- Begin with a hero similar to the homepage but with shop-specific headline and supporting copy.
- Use a pill-shaped category filter bar.
- Use a clean multi-column product grid on desktop.
- Keep mobile stacked but equally intentional.

**Newsletter / Contact Band**
- Use a large rounded horizontal band with bold heading, short copy, and email/form controls.
- The form should feel integrated into the pill/rounded visual language.

**Contact Page**
- Use a bold hero with the heart hands illustration.
- Use a large rounded split panel for the contact form and supporting illustration/product composition.
- Contact detail cards should use outlined rounded containers and playful icon circles.
- FAQ items should use slim outlined pill rows with colored circular expand controls.

**Footer**
- Use a dark footer with rounded top corners or large rounded container treatment.
- Include logo, navigation pills, social link, privacy, and terms.
- Continue the playful color shapes subtly around the footer edge if practical.

---
## 6. Typography Direction

Typography should feel bold, rounded, friendly, and graphic.

**Display Type**
- Use a chunky rounded display style for major headlines.
- Headlines should be large, condensed enough to feel punchy, and visually close to the reference.
- If the exact reference font is unavailable, choose the closest web-safe or licensed equivalent available during implementation.

**Body Type**
- Use a clean, readable sans-serif for paragraphs, prices, forms, and product detail text.
- Body text should remain calm and legible so the display type can carry the personality.

**Text Treatment**
- Use uppercase labels for nav, product names, badges, and small CTAs where it matches the reference.
- Maintain generous line height for body copy.
- Do not use thin, delicate, luxury-style typography.

---
## 7. Color Direction

The visual palette should be close to the references:

- Warm cream base
- Near-black text and outlines
- Bright royal blue
- Warm yellow
- Soft pink
- Orange
- Lavender
- Fresh green for product accents where appropriate
- Soft sky blue for secondary panels

Colors should be used as large rounded shapes, product tile backgrounds, small dots, line accents, and CTA highlights.

Avoid:

- muted beige-only layouts
- monochrome black-and-white-only layouts
- dark blue corporate palettes
- purple gradient SaaS styling
- generic ecommerce gray UI

---
## 8. Components

**Buttons**
- Prefer pill-shaped buttons with thick black outline or black fill.
- Primary buttons may use black fill with a colored circular arrow segment.
- Buttons should feel tactile and graphic.

**Product Cards**
- Use rounded color panels.
- Keep product cutouts/images centered and large.
- Use an outlined white/cream product info label at the bottom.
- Include price and a clear action affordance.

**Filters**
- Use a large pill segmented-control style.
- Keep filter count small for launch.
- Active filter should be visually obvious.

**Forms**
- Use rounded pill or rounded rectangle inputs with black outlines.
- Join buttons may use black fill.
- Validation states should remain clear and not depend only on color.

**Cart**
- Cart drawer or cart page should inherit the same rounded, outlined, playful language.
- Cart lines must remain readable and functional.

---
## 9. Decorative Language

Decorative elements are allowed and expected for this project.

Use:

- rounded blobs
- circles
- arches
- dotted groups
- hand-drawn connector lines
- small hearts
- black line accents
- rounded illustration frames

Do not use:

- generic gradient blobs
- SaaS-style glass cards
- stock-photo hero sections
- thin minimal luxury spacing
- sharp corporate panels
- decorative elements that cover text or product controls

Decorations should support the reference feel and should not make the site difficult to scan.

---
## 10. Proprietary Reference Rules

All files in `assets/reference/` are proprietary visual reference assets.

Agents may:

- Inspect all reference images locally.
- Match their layout, spacing, color energy, typography feel, and component style as closely as practical.
- Use them to judge whether the implementation is visually on target.

Agents must not:

- Import reference images into the production app.
- Use reference images as site imagery.
- Export or redistribute the reference images.
- Copy unrelated brand names, copy, logos, or proprietary imagery from the references.
- Treat reference images as source assets.

The intended output is a Hands Heart site that closely follows the reference direction while using Hands Heart-owned assets and content.

## 11. Design Acceptance Checklist

Before considering visual implementation complete, verify:

- The homepage feels close to `assets/reference/home.png`.
- The shop page feels close to `assets/reference/shop.png`.
- The about/story experience feels close to `assets/reference/about.png` if implemented.
- The contact experience feels close to `assets/reference/contact.png`.
- Any additional images in `assets/reference/` have been inspected and reflected where relevant.
- The heart hands drawing is prominent in the first viewport.
- The header uses a bold rounded outlined treatment.
- Product cards use rounded colored panels and outlined product labels.
- Bright accent colors are present and balanced.
- The base page background is warm cream or close to it.
- Typography feels bold, rounded, friendly, and graphic.
- Commerce actions remain clear.
- Mobile layout preserves the playful composition without overlap.
- Reference images are not shipped as production assets.
